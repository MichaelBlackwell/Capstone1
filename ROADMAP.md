# InsightForge — Capstone Roadmap

## Context
Build "InsightForge," an AI-Powered Business Intelligence Assistant for the Advanced Generative AI capstone. The app analyzes `sales_data.csv` (2,500 rows: Date, Product, Region, Sales, Customer_Age, Customer_Gender, Customer_Satisfaction) using LangChain + OpenAI + RAG, then presents insights via a Streamlit UI. Deadline is tight (1–2 weeks), so we prioritize core requirements first.

## Phase 1: Data Preparation & Knowledge Base (Days 1–2)

### Step 1 — Data Preparation (`src/data_loader.py`)
- Load `sales_data.csv` with pandas
- Parse `Date` column to datetime; extract `Year`, `Month`, `Day`, `Weekday`
- Verify data types, handle any nulls
- Compute pre-aggregated summaries:
  - Sales by time period (monthly, quarterly)
  - Sales by product
  - Sales by region
  - Customer demographics (age bins, gender splits)
  - Statistical measures: mean, median, std dev, min, max per group

### Step 2 — Knowledge Base Creation (`src/knowledge_base.py`)
- Convert aggregated summaries + raw data descriptions into text documents (one doc per analytical dimension)
- Use LangChain `Document` objects
- Embed documents using OpenAI Embeddings (`text-embedding-3-small`)
- Store in a FAISS vector store (lightweight, no external DB needed)
- Persist the vector store to disk for reuse

---

## Phase 2: LLM + RAG System (Days 3–5)

### Step 3 — Custom Retriever (`src/retriever.py`)
- Build a custom LangChain retriever that:
  - Queries the FAISS vector store for relevant summary docs
  - Also runs live pandas queries for specific numerical questions (e.g., "What are total sales in Q1?")
- This hybrid approach ensures the LLM gets both pre-computed context and precise numbers

### Step 4 — Prompt Engineering & Chains (`src/chains.py`)
- Design prompt templates:
  - **Summary prompt**: generates an executive summary of the data
  - **Q&A prompt**: answers user questions grounded in retrieved context
  - **Analysis prompt**: produces deeper analysis (trends, segmentation)
- Build a `RetrievalQA` chain (or `create_retrieval_chain`) linking retriever → prompt → LLM
- Use `ChatOpenAI` (gpt-3.5-turbo for cost efficiency; gpt-4 optional for quality)

### Step 5 — Memory Integration (`src/memory.py`)
- Add `ConversationBufferMemory` (or `ConversationSummaryMemory` for longer sessions)
- Wire memory into the chain so follow-up questions use prior context
- Use `ConversationalRetrievalChain` to combine RAG + memory

---

## Phase 3: Evaluation & Visualizations (Days 6–8)

### Step 6 — Model Evaluation (`src/evaluation.py`)
- Create a set of ground-truth Q&A pairs (10–15 questions with known answers from the data)
- Use LangChain's `QAEvalChain` to evaluate model responses against ground truth
- Report accuracy/correctness metrics

### Step 7 — Data Visualizations (`src/visualizations.py`)
- Build plotting functions using Plotly (interactive) or matplotlib:
  1. **Sales trends over time** — line chart (monthly/quarterly)
  2. **Product performance comparison** — bar chart by product
  3. **Regional analysis** — bar/map chart by region
  4. **Customer demographics** — age distribution histogram, gender split pie chart
  5. **Customer satisfaction** — box plot by product/region

---

## Phase 4: Streamlit UI (Days 9–11)

### Step 8 — Streamlit App (`app.py`)
- **Sidebar**: dataset overview, filters (date range, product, region)
- **Tab 1 — Dashboard**: key KPIs (total sales, avg satisfaction, top product/region) + auto-generated visualizations
- **Tab 2 — Chat with Data**: conversational interface powered by the RAG chain with memory; user types questions, gets NL answers + optional chart
- **Tab 3 — AI Insights**: one-click executive summary generated by the LLM
- **Tab 4 — Evaluation**: display QAEvalChain results showing model accuracy

## Risk Mitigation (tight timeline)

- **OpenAI costs**: use `gpt-3.5-turbo` by default; only switch to `gpt-4` for the report demos
